# Game Knowledge Loading

Use this skill at intake for `game_regulation` and `game_plus_general`.

## Principle

Game knowledge files are orientation material. They do not replace live primary
source verification and are not citable legal authority unless separately
verified as an official source.

## Selective Loading

Do not bulk-load all game knowledge. Load only the compact files that match the
query:

1. `knowledge/game-regulation/issue-taxonomy.md`
2. `knowledge/game-regulation/source-map.md`
3. `knowledge/game-regulation/library-index.md`
4. `knowledge/game-regulation/regulatory-map.md` when the query spans two or
   more jurisdictions or asks "who regulates this?"

## Freshness

For any knowledge file that includes `date_verified`:

- older than 6 months: use for orientation only and verify;
- older than 12 months: treat as potentially stale background;
- expired `valid_until`: do not rely on regulatory position.

Korean statutory positions must be verified against official current law before
being cited.

## Trust Tier Rules

- Tier 1, generated orientation: planning only; never cite.
- Tier 2, verified external summary: may guide issue spotting; verify before
  relying.
- Tier 3, attorney-curated: use only within its confidentiality and scope limits;
  do not quote internal work product into client-facing output unless marked
  shareable.

## Privacy Handoff

If a data-protection specialist is co-running, use privacy-oriented knowledge
only to identify the handoff facts. Do not perform deep PIPA/GDPR analysis.
