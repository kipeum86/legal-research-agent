# Game-Regulation Research

Use this skill in `game_regulation` mode and for the game-law core of
`game_plus_general`.

## Core Questions

Ask:

- What is the product: game, platform, marketplace, payment feature, or user
  generated content service?
- Which jurisdiction(s) matter?
- Are randomized rewards, paid items, virtual currency, or refunds involved?
- Are minors or age ratings involved?
- Are advertising, influencer marketing, dark patterns, or monetization
  disclosures involved?
- Are platform or app-store rules legally relevant?
- Is privacy merely a game-law handoff issue, or is a data-protection specialist
  running?

## Workflow Additions

At intake:

- apply `game-library.md`;
- load only matching compact game knowledge files;
- use knowledge files as planning orientation, not citable authority.

Before source collection:

- identify likely regulators using `knowledge/game-regulation/regulatory-map.md`
  when available;
- identify taxonomy categories from
  `knowledge/game-regulation/issue-taxonomy.md`;
- apply `jurisdiction-source-playbook.md` for jurisdiction profiles and source
  minimums;
- prepare jurisdiction-specific source plans.

## Adjacent Law Boundary

The following are normally part of `game_regulation` when tied to a game
product:

- consumer protection
- advertising
- youth protection
- platform/intermediary rules
- virtual items and paid currency
- payments and refund rules
- privacy spotting

Use `game_plus_general` only for distinct non-game legal issues.

## Privacy Handoff

If `co_running_agents` includes a data-protection specialist:

- do not analyze detailed PIPA/GDPR obligations;
- identify the game-law facts that trigger privacy review;
- mark privacy as a handoff issue in `issue_map`;
- state that privacy analysis is delegated to the co-running specialist.

## Multi-Jurisdiction Structure

For multi-jurisdiction game questions:

1. Provide jurisdiction-by-jurisdiction findings.
2. Keep game taxonomy categories visible for each jurisdiction.
3. Identify common compliance themes.
4. Include `comparison_matrix` when it clarifies differences.
5. State source limitations by jurisdiction.

Do not synthesize "global game law" before each jurisdiction has at least one
controlling or official source for every material launch, disclosure, procedure,
or enforcement issue.

## Output Expectations

- Use the issue taxonomy in `knowledge/game-regulation/issue-taxonomy.md`.
- Do not mimic a long legacy pipeline when a focused memo answers the question.
- Map each game-specific issue to source IDs.
- Distinguish official law/guidance from secondary commentary.
- Use a comparison matrix for multi-jurisdiction answers when it clarifies
  regulator, source, obligation, or enforcement differences.
- Include counter-analysis for gambling characterization, consumer-protection
  scope, enforcement uncertainty, and self-regulatory status where relevant.
