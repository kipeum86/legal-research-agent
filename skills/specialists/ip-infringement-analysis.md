# IP Infringement Analysis

## Runtime Rule

Use this file as the compact IP infringement analysis checklist. Load
`references/packs/ip-infringement-analysis.md` only when preparing an infringement
memo, pre-filing assessment, cease-and-desist support memo, licensing dispute
analysis, or settlement valuation.

## Prerequisites

Pause and ask for missing materials instead of filling gaps:

- IP registrations or filings
- accused product/service/activity materials
- prosecution history for patents
- licenses, NDAs, assignments, employment agreements, or restrictive covenants
- cease-and-desist letters, licensing negotiations, or admissions
- jurisdiction and procedural posture

## Execution Checklist

Read `references/packs/ip-infringement-analysis.md` and apply its detailed
frameworks, charts, defenses, remedies, troubleshooting, and guidance.

1. Identify each IP right separately.
2. Characterize the accused activity.
3. Apply the relevant framework:
   - Patent: claim construction, literal infringement, doctrine of equivalents, indirect infringement, validity.
   - Trademark: controlling likelihood-of-confusion test and dilution where applicable.
   - Copyright: ownership, access, substantial similarity, filtration, de minimis, fair use.
   - Trade secret: identification, secrecy value, reasonable measures, misappropriation method, covenants.
4. Assess defenses and vulnerabilities.
5. Evaluate remedies and damages categories.
6. State overall likelihood, confidence level, key vulnerabilities, and recommended course.

Analyze each IP type separately when multiple rights are at issue.

## Output Contract

The memorandum must include:

- IP rights identification table
- accused activity table
- type-specific infringement analysis
- claim chart when patent claims are at issue
- likelihood-of-confusion factor table when trademark claims are at issue
- fair-use table when copyright fair use is plausible
- trade-secret element analysis when trade secrets are at issue
- defenses and vulnerabilities table
- remedies assessment table
- overall assessment with confidence level

## Project Citation Rule

Use `[Unverified]` for any authority or citation not confirmed against source
documents. Do not use the legacy verify tag; it is not a project standard.

- [ ] Missing prerequisites are flagged instead of assumed.
- [ ] Each IP type is analyzed separately.
- [ ] Controlling jurisdiction and forum are identified.
- [ ] Patent analysis is claim-by-claim and limitation-by-limitation.
- [ ] Trademark analysis uses the controlling circuit's confusion factors.

- [ ] Trade-secret analysis identifies the alleged secret with specificity.
- [ ] Defenses and remedies are tied to the facts and procedural posture.
