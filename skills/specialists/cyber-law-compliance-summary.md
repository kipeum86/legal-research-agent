# Cyber Law Compliance Summary

## Runtime Rule

Use this file as the compact cyber/privacy compliance memo checklist. Load
`references/packs/cyber-law-compliance-summary.md` only when preparing a cyber law
compliance memo, privacy program assessment, incident-readiness review, or
compliance gap assessment.

## Prerequisites

Gather before drafting:

- business profile: industry, entity type, jurisdictions
- data inventory: categories, purposes, vendors
- existing policies: privacy policy, security program, incident response plan
- regulatory triggers: enforcement actions, audits, recent incidents

## Execution Checklist

Read `references/packs/cyber-law-compliance-summary.md` and apply its detailed
section templates, authority tables, jurisdiction comparison, sensitive-data, and
pitfall rules.

1. Draft executive summary with urgent requirements and risk priority matrix.
2. Cover applicable compliance sections.
3. Build jurisdiction comparison table when multiple jurisdictions apply.
4. Flag sensitive-data categories and heightened obligations.
5. Produce implementation checklist.
6. Include emerging trends and pending obligations.
7. Mark uncertain citations `[Unverified]`.

## Output Contract

The memo must include:

- executive summary
- compliance sections with legal requirement, business obligation, documentation, and penalty/enforcement treatment
- jurisdiction comparison table when applicable
- sensitive-data category analysis
- implementation checklist
- emerging trends section
- local counsel flags for non-US/EU or unsettled issues

## Quality Checklist

- [ ] Mandatory requirements are distinguished from recommended best practices.
- [ ] Each legal requirement has a citation or `[Unverified]` tag.
- [ ] Effective dates and pending deadlines are included.
- [ ] Jurisdiction-specific obligations are not collapsed into generic statements.
- [ ] Sector-specific regimes are flagged where applicable.
- [ ] Enforcement amounts and recent trends are included when material.
- [ ] Operational action items are concrete enough for implementation owners.
