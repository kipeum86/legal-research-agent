## AI Limitations

This project is powered by a large language model (Claude). LLMs can and
do produce **hallucinations**, **factual errors**, and **incorrect legal
analysis**. Outputs may:

- **Fabricate legal authorities** by citing statutes, regulations, case law,
  or agency guidance that do not exist;
- **Misstate the law** by presenting an outdated, repealed, or incorrectly
  interpreted rule as current and valid;
- **Conflate jurisdictions** by attributing a legal rule to the wrong
  country, state, or regulatory body;
- **Omit material issues** by failing to identify relevant regulations,
  exceptions, or pending legislative changes;
- **Misclassify source hierarchy** by treating non-binding guidance, blog
  commentary, or secondary summaries as primary legal authority;
- **Invent citations** by generating plausible-looking but entirely
  fictitious case names, docket numbers, statute references, or article
  titles.

The agent includes built-in safeguards — `skills/trust-boundary.md`,
`skills/source-collection.md` source-layer minimums,
`skills/source-grading.md`, `skills/claim-spot-check.md` source laundering
guard, `skills/claim-verification-loop.md`, `skills/currentness-check.md`,
`skills/quality-check.md`, and the standalone `/audit` citation auditor —
but **these safeguards reduce, not eliminate, the risk of error**. A
qualified human must always review and verify every output before any
reliance or action. The agent assists; the human decides.

## Data Handling

The agent makes LLM API calls, which means research queries and collected
source text are sent to the model provider (Anthropic). No data is stored
on external servers beyond the API request lifecycle under standard
commercial terms.

If you use this tool for matters involving confidential or privileged
information:

- Use a commercial API key with appropriate data-handling terms;
- Review [Anthropic's data usage policies](https://www.anthropic.com/policies)
  for the latest terms;
- Consult your organization's security and compliance requirements before
  processing sensitive material through any AI tool.

The agent's MCP and web-fetch surfaces also send queries to the configured
servers (e.g., the `korean-law` MCP server, `WebFetch`, `WebSearch`).
Treat any sensitive query accordingly.
